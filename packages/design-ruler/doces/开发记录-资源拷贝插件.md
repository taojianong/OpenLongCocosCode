### 模块架构
采用分层架构，保持逻辑独立性：

1. **ResourceCopyHandler类**：独立的资源拷贝处理器
   - 数据持久化：读写`resource-copy.json`
   - 拷贝逻辑：验证目录、过滤文件、递归拷贝
   - 目录选择：打开文件选择对话框
   - 待扩展：根目录管理、路径转换

2. **Main进程**：桥接面板和处理器
   - 接收IPC消息
   - 调用ResourceCopyHandler方法
   - 返回结果到对应的面板

3. **Panel UI**：用户界面（两个独立面板）
   - 设计标尺面板（index.ts）：标尺和对齐线管理
   - 资源拷贝面板（resource-copy.ts）：规则列表管理
    - 根目录设置界面
    - 路径类型显示和解析路径预览
    - 事件绑定和消息发送

### 面板架构
采用独立面板架构，两个面板完全独立：
- 设计标尺面板（`cocos-design-ruler`）：标尺和对齐线功能，使用 `panel` 字段配置
- 资源拷贝面板（`cocos-design-ruler.export`）：资源拷贝功能，使用 `panel.export` 字段配置
- 各自独立的消息通信和UI交互
- **面板命名规则**：
  - 主面板：`package-name`（如 `cocos-design-ruler`）
  - 附加面板：`package-name.xxx`（如 `cocos-design-ruler.export`）
  - 打开方式：`Editor.Panel.open(\`${PACKAGE_NAME}.export\`)`
  - 消息发送：`Editor.Ipc.sendToPanel(\`${PACKAGE_NAME}.export\`, ...)`

### 路径转换逻辑（已实现）

#### 设计原理
核心思路：**规则中尽量存相对路径，拷贝时动态解析为绝对路径**。这样更换根目录后无需修改规则即可正确拷贝。

1. **浏览目录时自动转相对路径**：用户通过"浏览"选择目录后，`openDirectoryDialog` 自动调用 `makeRelative()` 尝试将绝对路径转为相对路径：
   - 源目录：尝试相对化到 `sourceRoot`
   - 目标目录：尝试匹配 `exportRoots` 中的某个导出根目录，匹配成功则转相对路径

2. **拷贝时动态解析**：
   - `copySingleResource()` 根据 `targetDir` 路径类型决定拷贝行为：
     - **绝对路径**：直接拷贝到指定位置
     - **相对路径**：遍历所有 `exportRoots`，将相对路径拼接到每个导出根目录，逐个拷贝
   - 源目录始终通过 `resolvePath()` 解析：相对路径基于 `sourceRoot`，绝对路径保持原样

3. **面板预览解析结果**：`getResolvedPath()` 在面板上显示每条规则解析后的实际绝对路径，方便用户确认。

#### 关键方法
```typescript
// 绝对路径 → 相对路径（在根目录下时才转换）
makeRelative(filePath: string, rootPath: string): string

// 相对路径 → 绝对路径（拼接根目录）
resolvePath(rulePath: string, rootPath: string): string

// 判断是否为绝对路径
isAbsolutePath(p: string): boolean

// 执行目录拷贝（独立方法，可多次调用）
doCopyDir(rule: ResourceCopyRule, sourceDir: string, targetDir: string): void
```

### 使用方式

#### 设计标尺面板
1. 点击菜单"扩展/设计标尺/打开面板"
2. 设置对齐线颜色、透明度、样式
3. 添加水平/垂直对齐线
4. 导入设计图并调整透明度

#### 资源拷贝面板（当前）
1. 点击菜单"扩展/资源拷贝"打开面板
2. **设置根目录**（一次性设置）：
   - 点击"资源根目录"的"浏览"按钮，选择资源根目录
   - 点击"添加导出目录"，选择或添加多个导出根目录
3. **添加拷贝规则**：
   - 点击"+ 添加条目"
   - 点击"浏览"选择源目录和目标目录（自动转为相对路径）
   - 系统自动判断并显示路径类型徽章（蓝色=相对，红色=绝对）
   - 下方显示解析后的实际绝对路径预览
   - 选择拷贝规则（全部/文件名开头）
4. **执行拷贝**：
   - 勾选规则后点击"全部拷贝"或单独拷贝
   - 相对路径目标自动遍历所有导出根目录逐一拷贝

## 架构优势

### 逻辑独立
1. **数据隔离**：ResourceCopyHandler独立管理`resource-copy.json`
2. **功能解耦**：标尺功能和拷贝功能完全独立
3. **易于维护**：修改拷贝逻辑不影响标尺功能

### 可扩展性
1. **类封装**：ResourceCopyHandler可以独立扩展新功能
2. **接口清晰**：对外提供简单的方法接口
3. **便于测试**：独立模块易于单元测试

### 性能优化
1. **按需加载**：只有在切换到资源拷贝标签页时才执行相关操作
2. **事件驱动**：使用IPC消息机制，避免轮询
3. **懒加载**：资源拷贝逻辑按需初始化

### 路径管理（已实现）
1. **灵活配置**：支持相对路径和绝对路径自动切换
2. **团队协作**：规则存储相对路径，不同用户设置各自的根目录即可共享规则
3. **便携性**：更换根目录后无需修改规则，拷贝时动态解析
4. **自动转换**：浏览选择目录后自动转为相对路径（在根目录范围内）

## 注意事项

1. **模块独立**：ResourceCopyHandler应保持独立性，不依赖标尺相关的变量和函数
2. **命名空间**：使用`[resource-copy]`前缀区分日志，便于调试
3. **类型安全**：TypeScript严格模式下需要显式类型注解，避免使用保留字（如`path`）
4. **文件系统**：拷贝前检查目录存在性，使用`recursive`选项自动创建目录
5. **错误处理**：所有文件操作都需要try-catch包裹，避免插件崩溃
6. **编译配置**：tsconfig.json需要包含`resource-copy-handler.ts`
7. **路径规范**：Windows和Linux路径分隔符兼容性
8. **配置迁移**：需要提供从旧数据格式到新数据格式的迁移逻辑（如果需要）
9. **面板配置**：使用`panel`配置主面板，使用`panel.export`配置额外的面板
10. **面板命名规则**：附加面板名称使用点号分隔（如`cocos-design-ruler.export`），打开时使用`Editor.Panel.open(\`${PACKAGE_NAME}.export\`)`
11. **路径转义**：正则表达式中的反斜杠需要转义或使用双反斜杠
12. **数据持久化**：根目录配置单独保存在`resource-copy-config.json`中
