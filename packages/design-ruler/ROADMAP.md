# Design-Ruler 开发路线图

## 当前版本 (v2.2.0)

### 已实现功能
- 标尺刻度显示（自适应密度，随场景缩放动态调整间距）
- 标尺随场景缩放同步（通过 `_Scene.getEditingWorkspace().cameraSetting.operating2D.scale` 获取）
- 左上角实时显示当前缩放百分比
- 可拖拽的水平/垂直对齐线（支持从标尺区域拖出创建）
- 对齐线位置随场景缩放同步偏移
- 设计图叠加与透明度调整
- 通过 Canvas 注入实现的非侵入式渲染
- 悬浮控制面板（可拖拽移动、折叠）
  - 标尺显示/隐藏开关
  - 对齐线一键清除
  - 设计图加载/清除
  - 设计图透明度滑块
- ✅ **标尺坐标系与场景 (0,0) 精确对齐**（`originX = rect.left + xAxis`，`originY = rect.bottom + yAxis`）
- ✅ **标尺随场景平移同步**（顶部和左侧标尺均正确跟随）
- ✅ **对齐线超出编辑区域自动隐藏**（使用 `ctx.clip()` 裁剪到场景区域）
- ✅ **对齐线坐标数字不再与线重合**（水平线数字靠上，垂直线数字靠右，带背景块）

### 已知限制
- 仅支持 Cocos Creator 2.x
- inject.js 和 scene-script.js 使用 JavaScript，缺少类型安全
- 对齐线配置不持久化，重启后丢失
- 设计图仅支持单张叠加

**场景数据持久化实现思路：**
- 监听 `scene:ready` / `scene:enter-prefab-edit-mode` 事件获取当前场景 UUID
- 数据存储到 `packages/design-ruler/local-data/{sceneUuid}.json`，包含 `guides[]` 和 `designImagePath`
- 切换场景时先保存当前数据，再加载新场景对应数据
- 设计图存储路径引用（不存 base64），避免文件过大

## 长期愿景 (v3.0+)

### Cocos Creator 3.x 支持
- 适配 Cocos Creator 3.x 编辑器 API（扩展系统差异较大）
- 支持 3D 场景中的 2D UI 标尺
- 前置条件：Cocos Creator 3.x 扩展 API 稳定

### 响应式设计辅助
- 支持多分辨率预览和适配标注
- 显示不同设备尺寸的安全区域
- 自动计算 rpx/百分比换算

### 设计稿导入
- 支持从 Figma 直接导入设计标注数据
- 支持从 Sketch 导入设计标注
- 自动生成对齐线和参考线

### 协作功能
- 支持团队共享标尺配置（通过 Git 版本控制）
- 对齐线配置随场景文件一起提交

### 插件生态
- 提供公开 API 供其他插件集成标尺功能
- 支持自定义渲染层扩展

## 社区反馈收集

如有功能建议或 Bug 反馈，请通过以下方式参与：

1. 在项目 Issue 中提交功能请求，标注 `feature-request` 标签
2. 描述使用场景和预期行为，有助于评估优先级
3. 欢迎提交 PR 参与功能开发

优先级评估标准：
- **P0 紧急**：影响核心功能使用，需立即修复
- **P1 重要**：高频使用场景，显著提升体验
- **P2 期望**：有价值但非必须，按资源安排
- **P3 可选**：锦上添花，低优先级
